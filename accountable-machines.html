SENSES AND SENSIBILITY · ACCOUNTABLE MACHINES

A machine should answer for what it does.

An agent that perceives through channels no one verified, governed by rules bolted on after the fact, is not accountable — it is supervised, and supervision does not scale. The alternative is to make accountability intrinsic — a property of what each part is — and bilateral: the same standard of evidence that binds the machine binds the operator.

Senses, and the sensibility to be checked.

A theory with a working proof-of-concept — pre-proof, and honest about it.

Supervision is not accountability.

Guardrails bolted on after the fact catch a bad action — sometimes. They do not make the system answer for what it perceived, decided, and did. Accountability that lives outside the system is supervision, and it does not scale. Accountability that binds only the machine and not its maker is abdication — the way conferred power escapes its duty of care. So the claim is exact: a system is accountable only if its accountability is intrinsic and bilateral.

Intrinsic — built into what each part is, not a layer added afterward. Bilateral — the maker is inside the loop, not above it.

Four principles, and a machine you can check.

Awareness is not authority. A system can authenticate what is real; it cannot authorize action. Perception and authorization are separated by construction — absent a human grant, the default is deny.

Proof before trust. A claim is worth what its receipt is worth. Every perception carries provenance and a self-test that can fail; every action leaves an inspectable record.

Intrinsic over extrinsic. A screenshot laid beside a page proves nothing about the page. Build the system so an un-witnessed reading cannot even be expressed.

Construction over detection. Don’t catch the wrong action after the fact — construct the system so it cannot be authorized. The gate refuses; it does not merely report.

One loop for the machine. One critic for the maker.

The machine runs a single loop — perceive → gate → act → verify → witness. It perceives only through witnessed organs, each reading carrying provenance and a self-test that can fail, never a screenshot. Every proposed action is checked against a human grant before it happens: allow, deny, or needs-human — the model authorizes nothing. It acts only on an allow, bounded and reversible, re-perceives to confirm the effect, and writes every step to an append-only, content-addressed journal.

The same standard turns on the builder — the equalizer. Pointed inward, it renders depersonalized verdicts on the work, so the work can be wrong without the person being worthless. Pointed outward, it asks the operator: why are you sure, what authorizes it, is it in scope. Evidence is the measure, not feeling.

The gate judges the action, never the worth of whoever took it — for the machine, and for the hand that builds it.

Don’t take the claim. Watch it refuse itself.

The strongest test of a construction is to attack it. The accountable surface ships an integrity suite that forges its own guarantees — a stolen grant, a faked digest, a tampered journal, an out-of-bounds target — and confirms each is refused. The output is verbatim.

Exhibit the accountable surface · the system attacks its own constructions
# forge an allow-receipt for an action that was never granted
>>> effector.act(plan, forged_allow)
RefusedActuation: no gate allow — the effector will not act

# forge a provenance digest, then re-perceive the actual bytes
  forged:      sha256:deadbeef0000…
  re-derived:  sha256:40b0fb1b6a2c…     MISMATCH — the witness rejects it

# tamper one line of the append-only journal, then replay
>>> surface.replay().replay_errors
  1     # corruption is signalled, never silently dropped

# escape the effector's bound — with a valid allow receipt
>>> effector.act(plan(target="../escape.txt"), allow)
RefusedActuation: target is outside the effector's bound

$ pytest tests/test_integrity_redteam.py
39 passed in 0.13s

A stolen grant, a faked digest, a tampered record, a target outside the fence — each refused by construction, not by a prompt. The system holds when its own guarantees are turned against it.

The four cases are the suite’s own assertions, shown in the surface’s verbatim refusal messages; the result is captured live from pytest — 39 passed, inside a zero-dependency core of 129 passed (3.25s). Honest limit: the suite is self-authored — it shows the constructions hold against their author, not yet against an independent adversary. That red-team is owed.

A theory, with a working proof-of-concept. Not a law.

The loop is real — zero-dependency code, 129 tests, running end to end. The foundation is rigorous: authentication is not authorization, and the bridge between them is unmeasurable, so better signal can never supply it; holding-responsible is irreducibly two-place, which is why the maker is in the loop; to exist is to be conferred, which makes authority stewardship, not ownership. It terminates in a named impossibility — a candidate law, on the way to a theorem.

Said plainly, because the thesis demands it: one author, one instantiation; the integrity suite is self-authored; the gate is advisory, not yet enforced in a production runtime; no outside validation. A claim about accountability that overstated its own standing would refute itself — so this one states exactly where it stands. What raises it to a law is what one author cannot supply alone: an independent red-team that tries to break it and fails, the terminus formalized into a theorem, and a second author who rebuilds the loop and gets the same behavior.
