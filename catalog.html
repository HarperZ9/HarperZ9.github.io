The atlas: every release, and where it sits.

The whole body of work, mapped by its role. The eight public engines lead. Checkpoint, Runtime, Vault, and Boundary make the public-safe private-line layer visible. Under them sit the meatier substrates they plug into, then the small sharp tools, the bricks, and finally the papers and engine archaeology kept as repositories.

Flagships the eight that lead
telosthe engine, the perceive, generate, and verify surface. Demonstrated live in the Studio, whose generative surfaces are the Gallery (nineteen seed-authored plates plus a sound desk, one seed drawing a field, a solid, a face, and a phrase) and the Typeface builder (a seed authors a real font); animated sources record to WebM. Public demo repo at v0.2.0. Its telos proof command turns agent actions, research claims, scientific runs, and visual measurements into reviewable records.
indexthe repository map. Public, PyPI index-graph 2.8.0.
forumthe agent orchestrator. Routed work, causal ledger, and replayable decisions, v1.12.0.
gatherthe research-intake organ. Public, PyPI gather-engine 1.6.0.
cruciblethe judgment organ. Public repo; PyPI crucible-bench 1.1.0, release candidate, not claimed stable.
emetthe byte witness. Re-derives bytes and reports MATCH, DRIFT, or UNVERIFIABLE. PyPI emet v1.0.
buildlangthe typed-effects compiler. A verified C path plus shader output for authored, inspectable systems work. crates.io buildlang.
learnthe learning aid and credential organ. Witnessed, graded checkpoints on your own material, v1.5.0.
Private-line flagships public where safe
CheckpointAleph. Aleph coordinates docs, package contracts, MCP declarations, CI, supply-chain evidence, and receipts. Private line; the public overview is this page.
RuntimeORCA. Local run state, module execution, artifacts, release provenance, and owner handoff. Private line; the public overview is this page.
VaultKun. Path-only receipts, manual diagnostics, rotation discipline, and owner access boundaries. Private line.
Boundarybehavior-transform. Mode-aware read, write, exec, fetch, input, and model-boundary receipts. Public repo.
LabSeed. Private/proprietary controlled-lab assessment engine for synthetic fixtures, scenario packs, detection engineering, and receipt-backed validation.
LedgerSofer. Private/proprietary agent workflow ledger for action intent, admission, execution, evidence references, replay, and verification verdicts.
Secondary substrates the meatier repositories that plug in
proof-surfacethe check. An evidence-and-receipt contract that rejects authority-shaped claims.
coherence-membranethe perception organs. Inert, host-adjudicated reading of model context.
accountable-surfacethe live loop. A model perceives and acts under witness, check, and memory.
accountable-enginethe standard turned on the maintainer too. A critic that interrogates the human.
rawa real-time rendering research corridor, seed of a native renderer.
reconcilethe creative-verification engine. One operation over one substrate, every step witnessed.
studio-enginea zero-dependency simulation engine that generates and critiques artifacts.
mnemethe agent memory. Layered memory and hybrid retrieval; every entry carries provenance, every recall can be re-derived.
relaythe endpoint-agnostic coding agent. Local, subscription, and API endpoints behind one guarded tool loop with a replayable run ledger.
plexusthe interop mesh. Discovers what each tool emits and consumes, then wires producer to consumer; every edge cites its evidence.
build-colora color-science workbench: spaces, tone mapping, appearance models, ICC.
Bricks small, sharp, composable tools

The release and provenance tools, the agent tools, the kernels, the product line, and the small one-offs. Each is a utility you can adopt on its own. The shipping discipline behind them is the release toolkit.
