EMET

AI safety · Python / Rust / Node.js · MPL-2.0
A re-derivability-based integrity verifier for AI oversight: it checks that bytes reaching a model match their source and flags when a presented view differs from what should be read. Verdicts are MATCH, DRIFT, or UNVERIFIABLE.
3 implementations · conformance suite in CI · draft spec · STRIDE threat model

AI-safety research

Defensive work on model-facing proof and context integrity.

I write defensive notes on prompt and context-integrity failure modes in LLM coding assistants, including authorization-laundering / vocabulary-calibration as a safety-assessment pattern. The public primitive is EMET, a witness tool for source/view consistency. Operational exploit detail stays out of the public page by design.

Approach

My path is nontraditional: commercial arborist, Xbox networking support, then a sustained self-directed engineering sprint across compilers, graphics, color science, frontend sites, and AI tooling.

I use agentic coding tools heavily, but not as a substitute for ownership. I set the scope, review diffs, run tests, keep public/private boundaries clear, and label maturity honestly.
