ACCOUNTABILITY INFRASTRUCTURE · COHERENCE MEMBRANE

A guess about runtime state is not a perception.

A model’s one structural disability is state-blindness: it reasons on its prior and on source text, not on what actually happened. When it says “this frame renders correctly” or “this file contains X,” that is a guess dressed as a fact. The membrane replaces the guess with a witnessed, re-derivable Observation — full-width SHA-256, dimensions, perceptual hash — that the model can ground on. Inert by construction: it reads, never writes.

Drift verdicts in a closed lattice — MATCH / DRIFT / UNVERIFIABLE — never a silent match on a difference.

v0.1.0 · zero dependencies · 300+ tests · organ family: visual · raw · region · structured · audio · caption

State-blindness is structural, not a bug to patch.

The disability is not incidental — a model without a perception layer has no channel to runtime state. It simulates the world in its head. The membrane does not patch the model; it gives it a channel. Each organ reads an artifact — a PNG, a raw frame, a JSON document, a WAV, a caption — and returns an Observation carrying the bytes it witnessed: exact identity hash, organ-specific measurements, a perceptual fingerprint where the organ can derive one. The Observation is the evidence; the model grounds on the Observation, not on a simulation.

The safety and the capability are the same thing. A model that perceives true, re-derivable state cannot act on a hallucination about what is there (safety) and can close the make → look → compare → adjust loop it was always blind in (capability).

Inert organs: they read and report. They never mutate the artifact, the process that produced it, or anything else.

The organ family.

VisualArtifactOrgan — perceives a PNG into a full-width SHA-256, witnessed width and height, and a 64-bit dHash of the decoded pixels. The perceptual hash is the membrane’s drift signal: a cheap Hamming distance between two hashes measures how far the image moved, not just whether the bytes changed.

RawScreenCaptureSource / RawFrameOrgan — native screen capture via the OS’s own compositing API (GDI on Windows via ctypes, no third-party package). The raw fast path skips PNG encoding entirely: identity hash on the raw BGRA bytes every tick, perceptual hash only on a real change. Bit-identical to the PNG path, proven by selftest — same answer, lower cost.

RegionArtifactOrgan — the same whole-image facts plus a row-major grid of per-tile dHashes. compare_region_drift localises a change to the tiles that actually moved — “tile 5 changed” rather than “the whole screen changed.”

StructuredDataOrgan — perceives a JSON document into a raw-byte identity and a canonical identity (the digest of the document re-serialised in sorted-key, normalised-whitespace form). Baseline memory checks on a three-rung ladder: byte identity → canonical identity → perceptual distance. A reformatted-but-equivalent document is a MATCH; a changed value is a real DRIFT.

AudioArtifactOrgan — perceives a WAV (stdlib wave, no third-party audio stack) into identity, format, and a 64-bit perceptual fingerprint of the loudness envelope. Fails closed to identity-only on anything it cannot decode.

CaptionOrgan — perceives a subtitle or transcript line into a raw-byte identity and a canonical text identity (Unicode NFC and collapsed whitespace). A whitespace-different-but-same caption is a MATCH. Composed per-timestamp with a frame organ, the membrane witnesses what was on screen and what was said as one instant.

Two gates, deliberately separate.

The read-gate (this library) answers: what is actually there? It perceives real artifacts into witnessed Observations. The write-gate (proof-surface, pre_execution_gate) answers: may this action proceed, given that state? Default-deny, fail-closed, advisory.

They are separate repositories by design: a read-gate is useful to specifications that never act; a write-gate is useful to agents with no eyes. They compose through the shared observation and receipt JSON shape, not through a code dependency. A DriftVerdict’s MATCH / DRIFT / UNVERIFIABLE is the gate’s witness_verdict lattice, so perceived visual drift flows straight into the gate’s state check with no translation layer.

The membrane never grants authority and removes no check — every guarantee lives in the parts it composes.

Don’t take the observation. Watch it refuse to guess.

The membrane’s failure mode matters as much as its success mode. These cases show what happens when it cannot verify — and what happens when two implementations derive the same answer independently.

Exhibit I Coherence Membrane · drift as a closed lattice
# perceive a frame — witnessed identity, dimensions, perceptual hash
>>> snap = perceive(["frame.png"])
>>> obs = snap.observations[0]
obs.data["identity_sha256"]   # full-width, re-derivable
obs.data["width"], obs.data["height"]
obs.data["perceptual_hash"]   # 64-bit dHash

# drift check — closed lattice, never a silent match on difference
>>> compare_drift(baseline_sha, current_sha, baseline_phash, current_phash)
DriftVerdict(verdict="MATCH")    # byte-identical

>>> compare_drift(baseline_sha, changed_sha, baseline_phash, changed_phash)
DriftVerdict(verdict="DRIFT", distance=12)

# no pinned anchor — honest refusal, never a fabricated VALID
>>> verify_receipt(receipt)
ReceiptVerdict(verdict="UNVERIFIABLE", reason="no pinned anchor")

$ pytest
312 passed in current suite

The membrane reads, then reports. A match is a witnessed match. A missing anchor is UNVERIFIABLE, not a silent pass. The lattice is closed: no verdict outside MATCH / DRIFT / UNVERIFIABLE can be emitted.

Captured live from pytest — 312 passed (current suite, includes the live web-perception increment); stdlib-only, zero dependencies, github.com/HarperZ9/coherence-membrane. Formal lattice verification in lattice.py proves closure, fail-closed reachability, and monotonic attenuation by exhaustive enumeration — run on every pytest. Honest limit: organs are inert; a drift verdict is only as strong as the baseline it compares against. UNVERIFIABLE is returned rather than a guessed match. The perceptual hash is a coarse 64-bit fingerprint — advisory evidence, not semantic understanding.

What it is, and what it is not.

The membrane is inert perception infrastructure — stdlib Python, zero external dependencies, 300+ tests, organ selftests that can fail. It replaces a guess with a witness; it does not replace a gate. The write-gate (proof-surface) is where enforcement lives. The membrane reports; the operator adjudicates.

Screen capture reads the composited display output the operator can already see, via the OS’s own screen API through ctypes. It does not inject into, hook, or read another process’s memory. It must be used only on surfaces the operator owns or has authorised. That is not a disclaimer — it is the design.

The default is never affirmative — proven against the real functions, not asserted in a comment.

The formal lattice verification in lattice.py is machine-checked by exhaustive enumeration over the three-element verdict sets. The abstract laws are trivial for a three-element chain — deliberately. The work is the binding: executable proofs that the implementations land in, refine, and aggregate exactly as the lattice structures require.
