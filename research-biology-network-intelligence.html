Hyphal context protocol

The engineering hypothesis is simple: keep high-dimensional evidence in a durable substrate, move low-dimensional change signals through the tool graph, and retrieve evidence by receipt only when the receiver needs it.

  • Gather stores source bodies, refs, hashes, and availability state.
  • Index emits gradient envelopes for changed claims, freshness roots, missing links, and evidence pressure.
  • Forum routes by local path scores, verified outcomes, token cost, and reuse history.
  • Crucible classifies output claims as MATCH, DRIFT, or UNVERIFIABLE.
  • Learn turns the boundary into an operator lesson so the source packet becomes skill, not unsupported certainty.
  • BuildLang/buildc is the next runtime target for deterministic benchmark receipts.

Benchmark target

The next claim must be measured, not asserted.

Full-context route
Send all candidate source summaries into a model. Expected advantage: fewer retrieval misses. Failure mode: higher token cost and more unsupported synthesis.
Hyphal route
Send gradient envelopes plus receipt IDs, then retrieve evidence on demand. Expected advantage: lower token cost and stronger provenance joins. Failure mode: missing evidence if triggers are weak.
Measurements
Tokens spent, relevant evidence recovered, unsupported claims emitted, Crucible verdicts, resume quality after context loss, and whether later agents can join every claim to a receipt.
