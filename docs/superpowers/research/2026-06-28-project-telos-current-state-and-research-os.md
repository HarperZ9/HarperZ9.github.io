## Research OS Direction

The research model should not scrape everything into a single opaque pile. The rights-clean route is a multi-lane system:

- metadata spine first,
- rights/provenance gate second,
- full-text only when item-level reuse is allowed,
- excluded lane for sources that cannot enter production ingestion,
- hashes and source refs for every record that affects a model or agent decision.

The attached research report recommends a metadata spine across sources such as OpenAlex, Crossref, DataCite, OpenCitations, PubMed or Europe PMC, arXiv metadata, OSF, HAL, Zenodo, Figshare, and related repositories. Full text should be rights-clean and source-labeled, not silently imported from an uncheckable shadow path.

The local-model direction is a cognitive OS, not one large prompt:

- KV cache and memory systems for long-context efficiency,
- Index context envelope packets for lossless-by-reference codebase context,
- Gather receipts for source identity,
- Forum routing and prose clarification,
- Crucible measurement gates,
- Telos action receipts and loop ledger.

This avoids hidden payload or steganographic required context. Compression is allowed; unverifiable hidden meaning is not a dependency for enterprise code work.
