# Evidence-Led Portfolio Home Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the live portfolio lead with current recorded workflows and a direct paid-work route while preserving the existing dark generative workshop identity.

**Architecture:** Keep the Vite/React home as the source of truth and the existing static demo detail pages as the evidence layer. Add one asymmetric recorded-workflow rail that reads from a typed local data array, refresh the flagship version/maturity copy to the versions established by the current demo packages, and keep the interactive browser demonstrations as a secondary “try it here” section. Build through `home/deploy.mjs`, then test the generated root and publish the bounded commit to GitHub Pages.

**Tech Stack:** React 19, TypeScript 6, Vite 8, CSS, Python pytest static-contract tests.

## Global Constraints

- Preserve the existing `PRODUCT.md`, `DESIGN.md`, dark generative field, Telos Display/Kilon/Hanken Grotesk identity, WCAG 2.2 AA target, skip link, landmarks, and reduced-motion behavior.
- Range is the front door; verification supports the work but is not the brand thesis.
- Lead with real workflows, screenshots, repositories, transcripts, and maturity labels. Do not substitute social cards for native product evidence.
- Use the four verified public packages already present: Index 2.9.0, Gather 1.6.1, Forum 1.13.0, and Crucible 1.2.0.
- Link each workflow to its existing `/demo-*.html` detail page and poster under `/media/demos/`.
- Do not claim that Studio Engine, Mneme, Plexus, Relay, Flywheel, or Calibrate Pro has a new public recording until its same-run package is verified and published.
- No identical card grid, gradient text, decorative glassmorphism, repeated numbered-section scaffolding, or new tiny uppercase eyebrow system.
- Keep hero display size at or below 6rem and letter spacing no tighter than -0.04em.
- Body lines remain at or below 72ch; text content remains visible without IntersectionObserver or animation.
- Treat the email route already published on the site as the canonical direct contact; do not add new personal facts.
