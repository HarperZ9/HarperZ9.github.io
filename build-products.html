calibrate-pro 1.1: mature calibration core, legacy interface.

A Windows display-calibration toolkit for characterized and measured workflows. Its current PySide6 desktop application follows one bounded path: Detect → Method → Preview → Apply → Verify → Save/Report. It starts unelevated, previews the exact plan before a display write, and requires explicit confirmation before Apply; rejected, expired, substituted, or unsupported plans perform no write. The read-only CLI diagnostics cover installation and capability checks, calibration targets, characterized panels, stored evidence, HDR status, test patterns, and plugin discovery. Legacy direct-action names are proposal-only: they point to the GUI workflow instead of changing display state.

The core includes DDC/CI control where the monitor supports it, ICC/VCGT and LUT tooling, and native i1Display3 measurement support. Sensorless results are labelled estimated, not measured performance; a measured result requires a supported instrument, and missing observations remain Not measured. The calibration core and 1.1 packaging are current. The visual interface is still the legacy interface and is explicitly on the modernization track.

The color-science origin: years of film LUT pipelines and display work, packaged into a tool a person can actually run.

The forecasting & trading toolkits, paper-first by design.

build-oracle is a toolkit for predicting trends in data over time and spotting unusual breaks or jumps: ARIMA with automatic order selection, structural-break (changepoint) detection by information criterion (a score that picks the simplest explanation that still fits the data), and feature engineering, which builds extra signals like recent averages, lag values, and Fourier terms to help the model see patterns. The honest forecasting layer the others build on.

build-finance is a rule-based trading toolkit for stocks and crypto: backtesting (running a strategy against old data to see what would have happened), portfolio optimization using named methods (Markowitz, Black–Litterman, Hierarchical Risk Parity, Risk Parity), and strategies like an EMA-crossover momentum filter. It runs the analysis and the backtest; it doesn’t pretend a backtest is a promise.

build-engine is the self-adjusting loop on top: it trains forecasters, turns forecasts into signals, paper-trades or backtests them, and feeds the results back to adjust its own internal settings (the numbers that determine how the model behaves). It is paper-trading by default, and live brokerage is deliberately hard to switch on. It refuses to run live unless you set an explicit risk-acknowledgement in the environment. The guardrail is the feature.

These are real toolkits, honestly beta. Nothing here is financial advice, and the design assumes you’ll paper-trade first; it makes that the path of least resistance on purpose.

The suite, and one tool reporting itself.

build-ui is the shared visual style and controls library for the companion GUI tools. Calibrate Pro 1.1 uses the newer Build UI 2 integration with PySide6. build-ecosystem is the single package that installs the whole set in one command, and build-color is the color-science member, with its own page. The CLI makes inspection and discovery easy without presenting a direct-action calibration surface. Display mutation stays in the GUI’s preview, confirmation, prior-state capture, compensation, and authoritative read-back boundary.

Where they really are.
