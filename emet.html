The witness

WHAT IT IS

It re-derives the hash itself.

A model acts on bytes — a file it edits, a prompt it is handed, a view it presents as faithful to its source. EMET reads those raw bytes directly, recomputes the sha256 over them, and compares it to what was anchored. The verdict comes from re-running the same code on the same bytes, never from the system vouching for itself. It sits outside whatever it audits.

Build provenance proves where a file came from. It does not prove the bytes in front of the model still match it.
The lattice

THE VERDICTS

Three verdicts. None of them trusted.

The integrity lattice is closed by construction — exactly MATCH, DRIFT, UNVERIFIABLE. The spec declares it closed; the code makes that closure structural, not a matter of review. There is no path to an authority verdict, because the witness has none to confer.

MATCH the checked bytes re-derive to what was anchored · the absence of drift, never a grant of trust
DRIFT the bytes, or the presented view, no longer re-derive · reported as a fact, not adjudicated as a fault
UNVERIFIABLE no raw-byte channel, no anchor, no second read path · inability reported with a stable reason code — never a default, never a crash
The absence of DRIFT is MATCH or UNVERIFIABLE. It is never, by any path in the code, trust.

SEE IT HOLD

Don’t take the verdict. Watch it flip.

One byte decides everything. Verify a file, append a single byte, ask again — and the same spec, re-derived in three languages, agrees on the result.

Live the verdict depends on the bytes, nothing else
$ python membrane.py verify demo_sample.txt
MATCH  demo_sample.txt   want=d02206abca2a…  got=d02206abca2a…   (exit 0)

# append one byte to the file, ask the same question
$ python membrane.py verify demo_sample.txt
DRIFT  demo_sample.txt   want=d02206abca2a…  got=b072d611645e…   (exit 2)

Conformance

RE-DERIVED THREE TIMES

One spec, three languages.

The spec was implemented three times from its text: a stdlib-only Python reference, a from-scratch Rust build with no crates, and a clean-room Node.js implementation using built-ins only. All three pass the same nineteen conformance vectors on every push.

Three implementations agreeing is not three people agreeing.

They share one author. That earns exactly one claim and no more — the spec is implementable from its text. Independent re-derivation, a fourth implementation by a different hand, is owed; the roadmap says so plainly rather than letting the number imply it.

Run it

RUN IT YOURSELF

Clone it. Run the conformance. Use it on real work.

Small, open-source, dependency-light. Clone the repository, run the suite, point it at something that matters — and open an issue the moment it doesn’t hold. That feedback decides what gets built next.
